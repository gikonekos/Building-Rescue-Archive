# TRIG0 Analysis

Analysis of the routine `TRIG0` in Building Rescue v1.10.

## Conclusion

`TRIG0` is not a display synchronization routine.

It is a lightweight key-polling routine used by the title and ending loops.

## Code behavior

`TRIG0` scans the keyboard matrix through `KO` / `KI` and returns:

- `255` when the BASIC key condition is detected
- `1` when SHIFT is detected
- `0` otherwise

So its role is:

- quit handling
- title / ending trigger input

## Important implication

The pseudo-grayscale timing is not generated by `TRIG0` itself.

Instead, the grayscale effect is determined by the higher-level loop
structure, such as:

Title:
    VRAMPP
    VRAMPP
    VRAMPP2
    TRIG0

Ending:
    VRAMPP
    MPLAY
    VRAMPP2
    TRIG0

Therefore `TRIG0` should be treated as an input routine, not as a
synchronization routine.
